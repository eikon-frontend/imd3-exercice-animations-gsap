Lien vers le support de cours sur les animations GSAP

Qu'est-ce que GSAP ?

GSAP (GreenSock Animation Platform) est une bibliothèque JavaScript ultra-performante pour créer des animations fluides et professionnelles. Elle est considérée comme l'une des meilleures solutions pour l'animation web.

Animer un élément

La syntaxe de GSAP s’articule autour de quelques concepts.

// gsap.direction( qui, { comment } );

gsap.to(".box", {
  x: "-50px",
  duration: 2
});

Le paramètre qui accepte un sélecteur CSS (élément, classe, ID, pseudo-classe…) ou une variable ciblant un élément ou plusieurs éléments.

Les paramètres de l’animation sont compris entre accolades { … }. Il est possible d’animer plusieurs propriétés CSS du ou des éléments ciblés.

Méthodes de base

gsap.to() - Animer vers un état

// Animer un élément vers une nouvelle position
gsap.to(".element", {
  duration: 2,
  x: 100,
  y: 50,
  rotation: 45,
  scale: 1.5
});

gsap.from() - Animer depuis un état

// Animer un élément depuis une position initiale
gsap.from(".element", {
  duration: 1,
  opacity: 0,
  y: -50,
  scale: 0
});

gsap.fromTo() - Contrôle total

// Définir état initial ET final
gsap.fromTo(".element",
  { opacity: 0, scale: 0 }, // État initial
  { opacity: 1, scale: 1, duration: 1 } // État final
);

Propriétés animables courantes

Il n’existe pas de liste exhaustive des propriétés CSS pouvant être animées. La documentation de GSAP indique qu’une large partie des propriétés CSS peut l’être, le mieux étant de tester.

  • x, y : Translation sur les axes X et Y
  • rotation : Rotation en degrés
  • scale : Mise à l'échelle (1 = taille normale)
  • opacity : Transparence (0 = invisible, 1 = opaque)
  • width, height : Dimensions
  • backgroundColor : Couleur de fond
  • duration : Durée de l'animation en secondes

Timeline - Séquencer les animations

Il peut arriver de vouloir séquencer de multiples animations selon un timing précis.

Bien qu'il soit possible d'utiliser la propriété delay pour ceci, GSAP nous fournit un outil plus simple: gsap.timeline().

Une timeline est donc une sorte de container dans lequel on inscrit nos diverses animations selon un timing. 

// Créer une timeline
const tl = gsap.timeline();

// Ajouter des animations en séquence
tl.to(".element1", { x: 100, duration: 1 })
  .to(".element2", { y: 50, duration: 0.5 })
  .to(".element3", { rotation: 360, duration: 1 }, "-=0.5"); // Commence 0.5s avant la fin de la précédente
